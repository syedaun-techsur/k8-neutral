  <action>
Create `next.config.mjs` — MUST be `.mjs`, NEVER `.ts` (Next 14 hard-errors on TypeScript config — locked decision).

Write the EXACT content from TechArch verbatim:
```javascript
/** @type {import('next').NextConfig} */
const nextConfig = {
  async headers() {
    return [
      {
        source: '/(.*)',
        headers: [
          // No X-Frame-Options — allow iframe embedding
          // Do NOT add frame-ancestors to CSP
        ],
      },
    ];
  },
};
export default nextConfig;
```

IMPORTANT: The `headers` array is intentionally EMPTY. Do NOT add any X-Frame-Options header. Do NOT add a Content-Security-Policy with frame-ancestors. The purpose of this headers() function is to signal to Next.js that we are managing headers ourselves and to suppress any default X-Frame-Options that Next.js 14 might inject.

If a `next.config.ts` or `next.config.js` file exists, delete it — only `next.config.mjs` must exist.

After creating the file, run `npm run build` to confirm Next.js 14 can parse the `.mjs` config without error.
