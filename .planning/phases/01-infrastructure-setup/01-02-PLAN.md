Create `lib/db.ts` directory and file. Use EXACTLY the pattern from TechArch — copy verbatim, do NOT abstract or simplify:

```typescript
import { Pool } from 'pg';

if (!process.env.DATABASE_URL) {
  throw new Error('DATABASE_URL environment variable is required');
}

export const pool = new Pool({ connectionString: process.env.DATABASE_URL, max: 10 });

const migrate = pool.query(`
  CREATE TABLE IF NOT EXISTS notes (
    id         SERIAL PRIMARY KEY,
    title      TEXT        NOT NULL,
    body       TEXT,
    pinned     BOOLEAN     NOT NULL DEFAULT false,
    created_at TIMESTAMPTZ NOT NULL DEFAULT now()
  )
`).catch((err) => {
  console.error('[QuickNotes] Database migration failed:', err.message);
  throw err;
});

export const ready = migrate;

export function query(text: string, params?: unknown[]) {
  return pool.query(text, params);
}
```

**CRITICAL DDL — copy EXACTLY from TechArch, column names/types/constraints are canonical:**
```sql
CREATE TABLE IF NOT EXISTS notes (
  id         SERIAL       PRIMARY KEY,
  title      TEXT         NOT NULL,
  body       TEXT,
  pinned     BOOLEAN      NOT NULL DEFAULT false,
  created_at TIMESTAMPTZ  NOT NULL DEFAULT now()
);
```

Column-by-column verification:
- `id`: `SERIAL PRIMARY KEY` — auto-increment integer, no UUID
- `title`: `TEXT NOT NULL` — required, no max length constraint
- `body`: `TEXT` — optional (nullable), no NOT NULL
- `pinned`: `BOOLEAN NOT NULL DEFAULT false` — required with default
- `created_at`: `TIMESTAMPTZ NOT NULL DEFAULT now()` — timezone-aware timestamp

**Why this pattern works:**
- `pool.query()` is called at module load time (when `lib/db.ts` is first imported)
- This means the migration runs before any API route handler executes
- `export const ready = migrate` exposes the promise so API routes can `await ready` if they need to guarantee the table exists before querying
- The `catch` re-throws so if migration fails, the app fails loudly rather than serving broken responses

**Do NOT:**
- Use Prisma (not in stack — pg only per TechArch)
- Use `migrate.then()` chains — assign to `export const ready` as shown
- Add indexes — TechArch DDL has none for phase 1
- Add `updated_at` column — NOT in the canonical DDL
- Hard-code any connection string — only `process.env.DATABASE_URL`
